# ALU_with_UVM_based_Verification

## Overview

This project implements a **4-bit Arithmetic Logic Unit (ALU)** and verifies it using a complete **UVM (Universal Verification Methodology) testbench** built in **SystemVerilog**. The verification environment follows standard UVM architecture including sequence items, sequences, sequencer, driver, monitor, agent, scoreboard, environment, and test classes.

The ALU performs arithmetic and logical operations based on a 3-bit select signal, while the UVM testbench automatically generates randomized stimuli, drives them to the DUT, monitors outputs, and checks correctness using a reference model.

## Tools Used

* **Language:** Verilog + SystemVerilog + UVM
* **Simulation Platform:** EDA Playground

## Features

* 4-bit ALU design
* Full UVM-based verification environment
* Randomized stimulus generation using sequence items
* Functional coverage collection
* Reference model based scoreboard checking
* Automated PASS/FAIL reporting

## ALU Operations

| Select (S) | Operation              |
| ---------- | ---------------------- |
| 000        | Addition (A + B)       |
| 001        | Subtraction (A - B)    |
| 010        | Multiplication (A × B) |
| 011        | Division (A / B)       |
| 100        | Bitwise AND            |
| 101        | Bitwise OR             |
| 110        | Bitwise NAND           |
| 111        | Bitwise NOR            |

## DUT Inputs and Outputs

### Inputs

* `a [3:0]` → Operand A
* `b [3:0]` → Operand B
* `s [2:0]` → Operation Select

### Output

* `y [7:0]` → ALU Result

## UVM Testbench Architecture

The verification environment consists of:

* **Interface** → Connects DUT and testbench
* **Sequence Item** → Transaction object containing ALU inputs and outputs
* **Sequence** → Generates randomized transactions
* **Sequencer** → Controls transaction flow
* **Driver** → Drives inputs to DUT
* **Monitor** → Captures DUT inputs/outputs and collects coverage
* **Agent** → Groups driver, monitor, and sequencer
* **Scoreboard** → Compares DUT output with reference model
* **Environment** → Connects all components
* **Test** → Runs simulation and sequences

## Functional Coverage

Coverage is collected for:

* Operand A value ranges
* Operand B value ranges
* Operation select values
* Cross coverage between operands and operation modes

This helps ensure broad verification of ALU behavior.

## Verification Flow

1. Random transactions are generated by the sequence
2. Sequencer sends transactions to driver
3. Driver applies inputs to ALU
4. Monitor captures DUT outputs
5. Scoreboard compares DUT output with reference model
6. PASS/FAIL messages are displayed
7. Coverage report is generated at end of simulation
